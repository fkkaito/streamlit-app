# Streamlit TODO App
## Copilot アプリでの実装確認
PythonのWebフレームワークである **Streamlit** を使用して作成した、シンプルなTODOアプリケーションです。
HTMLやCSS、JavaScriptを一切使用せず、PythonコードのみでUI構築、状態（セッション）管理、およびデータの永続化を実現しています。

---

## 主な機能

### 基本機能
- **タスクの追加**: 入力エリアに文字を入力して「追加」を押すとタスクを登録できます。追加後は入力欄が自動でクリアされます。
- **ステータス管理**: チェックボックスを切り替えることで、タスクの「未完了 / 完了」の状態を直感的に更新できます。完了したタスクには自動で打ち消し線が表示されます。
- **タスクの削除**: 不要になったタスクを削除ボタン（🗑️）で一覧から除外できます。
- **データの永続化**: 登録したデータは、`app.py` と同じフォルダにある `todos.json` に自動保存されます。ブラウザのリロードや、Streamlitサーバーの再起動を行ってもデータは消えずに保持されます。

### 拡張機能（新機能）
- **優先度設定**: タスクに「高」「中」「低」の優先度を設定できます。優先度に応じたアイコン（🔴🟡🟢）で視覚的に表示されます。
- **期限設定**: タスクに完了期日を設定できます。期限を超過しているタスクには⏰アイコンが表示されます。
- **タスク検索**: キーワードでタスクを検索できます。タスク名に含まれるキーワードで絞り込み検索が可能です。
- **複合フィルター**: ステータスと優先度を組み合わせてタスクをフィルタリングできます。
- **完了タスク一括削除**: 完了したすべてのタスクをワンクリックで削除できます。
- **データエクスポート**: タスクデータをJSON形式でエクスポートでき、バックアップやデータ分析に活用できます。
- **優先度ソート**: タスク一覧は優先度（高→中→低）でソートされて表示されます。

## 実装の詳細

### 基本実装（Issue #3）

各タスクに対して以下の機能を実装しています：

1. **チェックボックスによるステータス更新**
   - TODOリストを`for`ループで処理
   - 各タスクを`st.checkbox`で表示
   - チェック状態が変更されると、該当タスクの`done`ステータスを更新
   - 完了したタスクは自動的に打ち消し線（~~テキスト~~）表示

2. **削除ボタン機能**
   - 各タスク右側に削除ボタン（🗑️）を配置
   - ボタンクリック時に該当タスクをリストから除外
   - `st.columns`を使用して、レイアウトを最適化

3. **即時UI更新**
   - 操作後に`st.rerun()`を呼び出してUIを即座に反映
   - ユーザーの操作体験を向上

### データ永続化（Issue #4）

1. **load_tasks()関数**
   - アプリ起動時にJSONファイルからタスクを読み込む
   - ファイルが存在しない場合は空リストを返す
   - エラーハンドリング対応

2. **save_tasks()関数**
   - タスク変更時にJSONファイルに保存
   - UTF-8エンコーディングで日本語対応
   - 成功時にTrue、失敗時にFalseを返却

### 拡張機能実装

1. **優先度・期限システム**
   - 各タスクに`priority`（高・中・低）フィールドを追加
   - 各タスクに`due_date`（期限日）フィールドを追加
   - 後方互換性を維持（既存タスクにデフォルト値を自動付与）

2. **検索・フィルター機能**
   - キーワード検索で複数条件の組み合わせが可能
   - ステータス、優先度の複合フィルター対応
   - リアルタイムでタスク一覧を更新

3. **UI/UXの向上**
   - `st.columns`でワイドレイアウトを実装
   - 優先度別のアイコン表示（🔴🟡🟢）
   - 期限のステータス表示（🔴超過 🟡本日 ⏰予定）
   - 優先度でのソート機能

4. **データ管理機能**
   - 完了タスク一括削除機能
   - JSON形式でのデータエクスポート
   - タイムスタンプ付きでのダウンロード

### コード品質改善

1. **エラーハンドリングの強化**
   - ファイル読み書き時の例外処理を追加
   - JSONデコードエラーやIOエラーに対応
   - ユーザーフレンドリーなエラーメッセージを表示

2. **セッション状態キーの一元管理**
   - 定数`SESSION_KEY_*`でセッションキーを集約
   - コード保守性と可読性の向上
   - キー名の誤字防止

3. **関数の分離と拡張性**
   - `load_tasks()`関数を実装（読み込み処理を分離）
   - `save_tasks()`関数を実装（戻り値で成功判定）
   - `export_tasks_json()`関数を実装
   - `get_priority_icon()`関数を実装
   - 型ヒントを追加（Python開発体験の向上）

4. **ドキュメント**
   - 各関数にドキュメント文字列を追加
   - コメントで処理を説明
   - サイドバーに使用方法を記載

## セットアップと起動手順

以下のステップに従って、ローカル環境でアプリケーションを起動できます。

### 1. プロジェクトフォルダの作成と移動

任意の場所にディレクトリを作成し、その中に移動します。

    mkdir streamlit-todo-app
    cd streamlit-todo-app

### 2. 仮想環境の作成と有効化

Pythonの仮想環境を作成し、有効化します。

**Windows (コマンドプロンプト):**

    python -m venv venv
    venv\Scripts\activate

**macOS / Linux:**

    python3 -m venv venv
    source venv/bin/activate

### 3. ライブラリのインストール

`streamlit` をインストールします。

    pip install streamlit

### 4. `app.py` の作成

プロジェクトのルートディレクトリに `app.py` という名前でファイルを新規作成し、後述する「ソースコード例」のコードをすべて貼り付けて保存してください。

### 5. アプリケーションの起動

以下のコマンドを実行してアプリケーションを起動します。

    streamlit run app.py

コマンド実行後、自動的にブラウザが立ち上がり、 `http://localhost:8501` にてアプリの操作が可能になります。
